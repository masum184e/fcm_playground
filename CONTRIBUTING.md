## How Can I Contribute?

You can contribute by:
- Reporting bugs 
- Suggesting new features 
- Improving documentation 
- Submitting pull requests 

## Reporting Bugs

Before creating a bug report:
- Check the existing issues to avoid duplicates
- Make sure you are using the latest version of the project

When creating an issue, include:
- A clear and descriptive title
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Screenshots or logs (if applicable)
- Environment details (OS, Android version, Firebase version, etc.)

## Suggesting Enhancements

Enhancement suggestions are welcome!

Please include:
- A clear use case
- Why the feature would be useful
- Possible implementation ideas (optional)

## Development Setup

1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/<your-username>/fcm_playground.git
   ```

## Commit Message Guidelines

Use clear and consistent commit messages:

Examples:
```
feat: add FCM background message handler
fix: resolve notification click crash
docs: update README setup instructions
```

## Pull Request Guidelines

Before submitting a pull request:

- Ensure the project builds successfully
- Follow existing code style and structure
- Add or update documentation if needed
- Test your changes

Pull Request rules:

- One feature or fix per PR
- Use clear and descriptive PR titles
- Reference related issues (e.g., Fixes #12)

Example PR title:
```
Add background notification handling
```
